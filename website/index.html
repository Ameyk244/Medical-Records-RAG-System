Differentiators

Why this is different

Six guarantees we built into the architecture, not the marketing.

Grounded answers only

If retrieved context doesn't support an answer, the system refuses. Not a feature toggle. A hard rule enforced in the synthesis prompt.

Forensic-grade audit

Every query and response written to an append-only Postgres table. A database trigger refuses UPDATE and DELETE — even from a superuser session.

Native multi-format

PDF, FHIR R4, HL7 v2, CSV out of the box. No ETL pipeline. New parsers add without schema changes.

Patient-scoped retrieval

Every search is scoped to one patient_id. Cross-patient leakage isn't a policy. It's architecturally impossible.

Sub-2s semantic search

HNSW index on pgvector with voyage-3-large embeddings. Millions of chunks, single-digit-ms recall.

Deploy anywhere

Docker Compose for on-prem (HIPAA requirement for many hospitals). Same codebase runs cloud-native. No vendor lock-in.

Security

Built for regulated environments

HIPAA-aligned by architecture, not by audit checklist.

Healthcare security cannot be bolted on after deployment. Every design decision in this system — from the append-only audit log enforced by a database trigger, to patient-scoped vector retrieval, to credentials flowing exclusively through environment variables — was made with a regulated environment as the baseline requirement.


This means no data leaves your perimeter if you deploy on-prem. No cross-patient retrieval path exists in the code. And no query is ever executed without first passing through the audit agent's access check — a hard architectural rule, not a middleware toggle someone can disable.

  • JWT bearer auth, bcrypt password hashing
  • Role-based access enforced before retrieval
  • DB-trigger immutable audit log
  • Patient-scoped queries (no cross-patient retrieval path)
  • Encryption-at-rest on sensitive columns (pgcrypto)
  • All credentials via env vars, never source
  • Deploy on-prem in your VPC — no data leaves your perimeter

Audit Agent

Runs before retrieval (access check) and after synthesis (append-only response log). Cannot be skipped.

Query Agent

Embeds the query via Voyage AI, runs pgvector similarity search filtered to one patient_id, returns top-5 chunks.

Synthesis Agent

Calls Claude with retrieved chunks as the only context. Refuses to answer if context doesn't support the claim.

Ingestion Agent

Triggered on upload. Parses document format, chunks at 512 tokens with 50-token overlap, embeds, upserts to pgvector.
